# eval/ — change-review AI eval harness

Offline eval for the AI pipeline described in [docs/CHANGE-REVIEW.md](../docs/CHANGE-REVIEW.md) §11.
This is a **dev tool** — it is not shipped in the npm package (see `files` in
`package.json`), not run by `npm test` (which is deterministic + offline), and
depends on `@anthropic-ai/sdk`, a `devDependency`.

## Run it

```sh
export ANTHROPIC_API_KEY=sk-ant-...
npm run eval                 # all fixtures
npm run eval insert-section  # one fixture
```

It sends each fixture's before/after/diff screenshots + PR intent to
`claude-opus-4-8`, forces the structured output in [schema.ts](schema.ts), and
scores it against the fixture's `expected.json`. Exit code is non-zero if any
fixture fails.

## What it checks (per docs/CHANGE-REVIEW.md §11.2)

- **Gate** — did the model classify the page correctly (analyze / refuse / triage)?
- **Regions** — did it find the real changes, with the right verdict/changeType?
- **Citations** — every region must carry a non-empty before/after citation
  (the "no citation, no region" rule §9).
- **Missing** — did it detect promised-but-absent claims?
- **Fabrication guard** — `mustNotContain` lists content that does NOT exist in
  the fixture. If the model names any of it, the case **fails hard**, regardless
  of every other score. This directly regression-tests the "Article cards"
  failure from the live report.

## Fixtures

Each fixture is a directory under `fixtures/`:

```
fixtures/<name>/
  base.png  head.png  diff.png   before / after / pixel diff
  intent.json                    { title, body, comments }
  expected.json                  ground-truth labels (see schema.ts → Expected)
```

The bundled fixtures are synthetic block-rendered pages generated by
[make-fixtures.ts](make-fixtures.ts) (`npm run eval:fixtures`) so the repo stays
self-contained. To add a **real-site** fixture, drop real `base/head/diff.png`
plus `intent.json` and `expected.json` into a new `fixtures/<name>/` — no code
change needed.

### The case set (grows with the envelope)

Current: `hero-copy-edit` (in-place text), `insert-section` (reflow —
movement must not be flagged), `promised-but-absent` (missing detection). Add
`full-redesign`, `wrong-baseline`, `viewport-mismatch`, `restyle-button`, and
`new-page` as the envelope widens (docs/CHANGE-REVIEW.md §11.4).

## Determinism caveat

The model is not bit-deterministic, so this is a scored/threshold suite, not an
exact-match unit test. Expect the occasional flake on borderline fixtures;
`mustNotContain` guard failures are the release-blocking signal.
